# Goal 模式重设计 P0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 Goal 模式在无人值守下「安全」——加 run 级 token/时间预算护栏(超预算强制停),并让模型能显式自报完成(`complete_goal` 工具),裁判 LLM 降为兜底且失败不再静默放行。

**Architecture:** `config.goal` 从 `string` 升为 `GoalConfig`(`{ objective, tokenBudget?, timeBudgetMs? }`)的契约升级,贯穿 turn-loop / protocol / engine / server。TurnLoop 维护一个 run 级累加器(累计 prompt+completion token + 墙钟起点),每轮结束结账,超预算返回新 `TerminalReason: "goal_budget_exhausted"`。新增 `complete_goal` 内建工具:模型调用即在 turn-loop 短路成 `completed`,绕过裁判;裁判 hook 仅在「模型没自报、又想停」时兜底,且解析失败改为注入续跑提醒(仍在预算内)而非放行。

**Tech Stack:** TypeScript (bun)、Zod(settings/protocol 校验)、bun test(TDD)。`@cjhyy/code-shell-core` 包。

**关键约束:**
- 向后兼容:旧调用方仍可传 `goal: "..."`(string)。在最外层入口(protocol/server.ts、engine.ts run 入口)用一个 `normalizeGoal()` 把 `string | GoalConfig | undefined` 收敛成 `GoalConfig | undefined`。**内部一律用 `GoalConfig`**,不在 7 处各写 union。

```typescript
// packages/core/src/engine/goal.ts
/**
 * Goal mode P0 — run-scoped budget guardrails.
 *
 * `GoalConfig` is the normalized internal shape. Public entry points
 * (protocol/server, engine run) accept `string | GoalConfig` for
 * back-compat and call `normalizeGoal()` once at the boundary; everything
 * inward uses `GoalConfig`.
 *
 * The budget tracker is RUN-scoped (not per-turn like token-budget.ts):
 * it accumulates total tokens across turns and stamps a wall-clock start,
 * so an unattended goal run can't burn tokens or wall time without bound.
 */

export interface GoalConfig {
  /** The objective text shown to the judge / injected into context. */
  objective: string;
  /** Hard cap on total tokens (prompt+completion) for the whole goal run. */
  tokenBudget?: number;
  /** Hard cap on wall-clock duration of the whole goal run, in ms. */
  timeBudgetMs?: number;
}

/**
 * Coerce a raw goal input into a normalized GoalConfig, or undefined when
 * there is effectively no goal (empty objective). Drops non-positive
 * budgets (0 / negative → "no limit", same as absent).
 */
export function normalizeGoal(raw: string | GoalConfig | undefined): GoalConfig | undefined {
  if (raw == null) return undefined;
  const obj: GoalConfig = typeof raw === "string" ? { objective: raw } : { ...raw };
  const objective = (obj.objective ?? "").trim();
  if (!objective) return undefined;
  const out: GoalConfig = { objective };
  if (typeof obj.tokenBudget === "number" && obj.tokenBudget > 0) out.tokenBudget = obj.tokenBudget;
  if (typeof obj.timeBudgetMs === "number" && obj.timeBudgetMs > 0) out.timeBudgetMs = obj.timeBudgetMs;
  return out;
}

export interface GoalBudgetTracker {
  goal: GoalConfig;
  /** Wall-clock start (ms epoch), captured at run start. */
  startedAtMs: number;
  /** Accumulated total tokens (prompt + completion) across all turns. */
  tokensUsed: number;
}

/** Create a run-scoped tracker. `nowMs` is injected for testability. */
export function createGoalBudgetTracker(goal: GoalConfig, nowMs: number): GoalBudgetTracker {
  return { goal, startedAtMs: nowMs, tokensUsed: 0 };
}

/** Add this turn's token usage to the running total. */
export function recordGoalUsage(tracker: GoalBudgetTracker, turnTokens: number): void {
  if (turnTokens > 0) tracker.tokensUsed += turnTokens;
}

/**
 * Has the run exceeded any configured budget? `nowMs` is injected so callers
 * pass the current clock (and tests pass a fixed value).
 */
export function goalBudgetExceeded(tracker: GoalBudgetTracker, nowMs: number): boolean {
  const { tokenBudget, timeBudgetMs } = tracker.goal;
  if (typeof tokenBudget === "number" && tracker.tokensUsed > tokenBudget) return true;
  if (typeof timeBudgetMs === "number" && nowMs - tracker.startedAtMs > timeBudgetMs) return true;
  return false;
}
```

```typescript
 * complete_goal — Goal mode P0.
 *
 * The model calls this to DECLARE the goal complete (mirrors Codex
 * update_goal(status=Complete)). When the turn loop sees this tool call it
 * short-circuits to a "completed" stop WITHOUT running the judge LLM. The
 * tool itself just records an acknowledgement into the transcript; the
 * loop-level short-circuit lives in turn-loop.ts (Task 4).
 *
 * NOTE: field names (name/description/inputSchema/handler/permissionDefault)
 * MUST match the BuiltinTool interface confirmed in Step 0. The skeleton
 * below uses the common shape; align it to the real interface.
 */
import type { BuiltinTool } from "./index.js"; // ← confirm the real type/path in Step 0

export const completeGoalTool: BuiltinTool = {
  name: "complete_goal",
  description:
    "Declare the current goal complete. Call this ONLY when the goal is fully achieved. " +
    "After calling, you should stop. Optionally pass a one-line summary of what was accomplished.",
  permissionDefault: "allow",
  inputSchema: {
    type: "object",
    properties: {
      summary: {
        type: "string",
        description: "Optional one-line summary of what was accomplished.",
      },
    },
    required: [],
  },
  handler: async (input: { summary?: string }) => {
    const summary = typeof input?.summary === "string" ? input.summary.trim() : "";
    return {
      // Match the real tool-result shape from Step 0. The key requirement:
      // the textual content says the goal is marked complete (so it lands
      // in the transcript) and includes the summary if provided.
      content: summary
        ? `Goal marked complete. Summary: ${summary}`
        : "Goal marked complete.",
    };
  },
};
```
