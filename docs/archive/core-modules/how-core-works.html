core 是无界面的引擎包,被 tui 与 desktop 共享。约 32 个模块、~64k 行。下面先看分层地图,再让一个数据包沿主干流水线走一遍 —— 步骤与高亮均对应真实代码路径。

PART 02主干流水线 · 一轮 turn loop

数据包从用户输入出发,沿 protocol → engine → prompt/context → llm → tool-system 流动。模型若要调工具,就钻进下面那条「工具闸道」,执行完回灌给 llm 再跑一轮;直到模型不再要工具,session 落盘、事件流回宿主。

工具闸道 · ToolExecutor.executeSingle()

tool-system 内部:模型的一次 tool call 要依次过这一串关卡,全过了才真正执行,然后再过一遍 post 钩子。

门控 能力/计划/MCP→ 校验 JSON schema→ pre 钩子→ 路径策略 PathPolicy→ 权限 分类/审批→ 执行 沙箱+超时→ post 钩子
就绪 点「开始演示」后,跟着高亮看一个请求如何穿过 core 的各层。每一步都对应仓库里真实的调用链。
对应源码 · engine/turn-loop.ts(turn loop)· protocol/server.ts(run 入口)· context/manager.ts(控窗口)· llm/client-factory.ts(模型调用)· tool-system/executor.ts(闸道)· session/session-manager.ts(落盘)。
每个模块的「职责/文件/公开接口/怎么用/注意」详见同目录 core-modules/<模块>.md,总览见 README.md。本动画为示意,步骤经过简化但顺序与真实 dispatch 一致。
