# data

**One-line role.** Bundled JSON model catalogs (OpenRouter snapshot + hand-maintained per-provider tables) plus the lazy loaders that read them at runtime.

## 职责 / Responsibility

This module ships static model metadata — context window, max output tokens, input/output pricing, modalities — as JSON assets and exposes typed accessors over them. It exists because many providers' `/v1/models` endpoints return only model ids with no metadata; the rest of `core` calls the live endpoint to discover ids, then left-joins these tables on `id` to fill in the missing fields. Boundaries: it does not fetch live model lists itself (that's `llm/model-fetcher.ts`) except for the one explicit `/sync-models` runtime refresh in `openrouter-sync.ts`. It is a read-mostly data layer; the only mutation is an in-memory OpenRouter snapshot override.

## 文件 / Files

| File | Purpose |
| --- | --- |
| `static-catalogs.ts` | Loader for hand-maintained per-provider tables; maps `ProviderKindName` → catalog JSON. |
| `openrouter-models.ts` | Lazy accessor over the bundled OpenRouter snapshot; supports an in-memory override. |
| `openrouter-sync.ts` | Runtime fetch of `openrouter.ai/api/v1/models`; replaces the in-memory snapshot (powers `/sync-models`). |
| `openrouter-models.json` | ~115KB build-time snapshot of OpenRouter's full catalog (generated by `scripts/sync-models.ts`). |
| `deepseek-models.json` | Static DeepSeek catalog. |
| `gemini-models.json` | Static Google/Gemini catalog (loaded under kind `google`). |
| `openai-models.json` | Static OpenAI catalog. |
| `zai-models.json` | Static Z.AI catalog. |

JSON shape (all catalogs): `{ fetchedAt, source, count, models: StaticModel[] }`.

## 怎么用 / How to use

Enriching a live model list with static metadata, then filling remaining gaps from the OpenRouter snapshot (this is exactly what `llm/model-fetcher.ts` does):

```ts
import { listStaticModels, hasStaticCatalog } from "../data/static-catalogs.js";
import { findOpenRouterModel } from "../data/openrouter-models.js";

function enrich(kind: ProviderKindName, live: CachedModel[]): CachedModel[] {
  if (!hasStaticCatalog(kind)) return live;
  const staticById = new Map(listStaticModels(kind).map((m) => [m.id, m]));
  return live.map((m) => {
    const stat = staticById.get(m.id);
    if (!stat) return m;
    // live values win when present; static only fills zeros
    return {
      id: m.id,
      contextLength: m.contextLength > 0 ? m.contextLength : stat.contextLength,
      maxOutputTokens: m.maxOutputTokens > 0 ? m.maxOutputTokens : stat.maxOutputTokens,
    };
  });
}

// Native providers return bare ids; the OpenRouter snapshot keys on "<vendor>/<id>".
const hit = findOpenRouterModel(`anthropic/${someBareId}`);
```

Pricing lookup against the snapshot (from `cost-tracker.ts`):

```ts
import { findOpenRouterModel } from "./data/openrouter-models.js";

const hit = findOpenRouterModel(model)                  // "vendor/model" form
  ?? findOpenRouterModel(`deepseek/${model}`)           // or probe by bare name
  ?? findOpenRouterModel(`anthropic/${model}`);
if (hit && (hit.inputPricePerMillion > 0 || hit.outputPricePerMillion > 0)) {
  // use hit.inputPricePerMillion / hit.outputPricePerMillion
}
```

Refreshing the catalog at runtime (the `/sync-models` slash command):

```ts
import { syncOpenRouterCatalog } from "@codeshell/core"; // re-exported from index
const result = await syncOpenRouterCatalog();
if (!result.ok) console.error(result.error);
```

## 注意 / Gotchas

- **JSON is loaded via `createRequire(import.meta.url)`, not `import`.** This module is ESM but reads `.json` through a CJS-style `requireJson`. The `.json` files must sit beside the compiled output in `dist/data/` — `package.json` build / asset copying must include them, otherwise loaders silently get nothing.
- **Empty-snapshot is a valid state, not an error.** On a fresh checkout before the first build (or a failed sync with no prior file), the OpenRouter accessors return `[]`. Every consumer must fall back to hardcoded defaults — never assume `getOpenRouterModels()` is non-empty.
- **All enrichment is fill-zeros-only.** Live `/v1/models` values always win; static/snapshot data only fills fields that are `0` or missing. Don't invert this.
- **OpenRouter ids are namespaced `<vendor>/<model>`; native catalogs use bare ids.** To cross-reference a native model against the snapshot you must prepend the vendor prefix (and the vendor name isn't always the kind name — e.g. zai → `z-ai`, xai → `x-ai`, mistral → `mistralai`).
- **`setOpenRouterSnapshot` / `syncOpenRouterCatalog` mutate process-global in-memory state and do NOT persist.** A sync lasts only for the current process; to make it permanent you re-run the build (`scripts/sync-models.ts` regenerates `openrouter-models.json`). OpenRouter is deliberately NOT in the static-catalogs table because its endpoint already exposes full metadata.
- **`static-catalogs.ts` table is keyed by `ProviderKindName`** and only covers `deepseek`, `zai`, `openai`, `google` (gemini JSON loads under `google`). Adding a provider means adding both a JSON file and a `TABLES` entry — and the metadata is hand-maintained, so it goes stale; edit the JSON when a vendor ships a new model.
